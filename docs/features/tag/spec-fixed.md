# 태그 기능 정의서 (확정)

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 기능 요구사항

- 노트에 태그를 추가할 수 있다.
- 노트에 추가된 태그를 삭제할 수 있다.
- 태그 목록을 노트 상세 화면에서 확인할 수 있다.

---

## 데이터 구조

### Note 타입 변경

`tags: string[]` 필드를 추가한다.

```ts
interface Note {
  id: string;
  title: string;
  content: string;
  tags: string[];       // 추가
  createdAt: string;
  updatedAt: string;
}
```

- 기본값: `[]` (빈 배열)
- 태그는 문자열 배열로 Note 내부에 저장된다.
- `db.json`의 notes 컬렉션에 `tags` 필드가 포함된다.

## UI

### 위치

`NoteEditor` 내 제목 입력 바로 아래, 본문 입력 위에 배치한다.

```
[ 제목 입력창                    ]
[ tag-a ]  [ tag-b ]  [ 태그 입력... ]
[ 본문 입력창                    ]
```

### 태그 칩 (pill)

- 추가된 태그는 칩(pill) 형태로 가로로 나열한다.
- `×` 버튼은 칩에 hover 시에만 표시된다.
- `×` 클릭 시 해당 태그를 즉시 삭제하고 서버에 반영한다.

### 태그 입력

- 칩 우측에 인라인 텍스트 입력창을 배치한다.
- `Enter` 키 입력 시 태그를 추가하고 입력창을 초기화한다.
- 입력창은 placeholder로 `태그 추가...`를 표시한다.

## 동작 규칙

### 입력 처리

| 조건 | 동작 |
|------|------|
| 앞뒤 공백만 있는 입력 | trim 후 빈 문자열이면 무시 |
| 이미 존재하는 태그 (대소문자 구분, 완전 일치) | 조용히 무시 (에러 표시 없음) |
| 최대 20자 초과 | 무시 (입력 자체를 20자로 제한) |

- 태그는 trim() 후 저장한다.
- 중복 판단은 대소문자를 구분한다 (`React`와 `react`는 별개).
- 허용 문자 제한 없음 (한글, 영문, 숫자, 특수문자 모두 허용).

### 저장 시점

- 태그 추가 및 삭제 즉시 `PATCH /notes/:id`로 서버에 반영한다.
- title/content 저장 버튼과 독립적으로 동작한다.
- 저장 실패 시 에러는 기존 `NoteEditor`의 에러 처리 방식을 따른다 (try/catch → 호출부 전파).

## 결정 사항 (임의 결정)

| 항목 | 결정 | 이유 |
|------|------|------|
| 태그 최대 길이 | 20자 | 칩 UI 레이아웃 고려 |
| 태그 수 제한 | 없음 | 스펙 미정의, 단순하게 유지 |
| 대소문자 구분 | 구분함 | 명시적 의도 존중 |
| 중복 시 피드백 | 없음 (조용히 무시) | UX 노이즈 최소화 |
| 태그 필터링 기능 | 미포함 | 원본 스펙 범위 외 |
| `×` 버튼 노출 조건 | hover 시만 표시 | 사용자 요청 |
