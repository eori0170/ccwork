# PRD: 태그 기능

> 상태: 초안
> 참조 스펙: `docs/features/tag/spec-fixed.md`
> 기술 결정(ADR): 다음 세션에서 논의 예정

---

## 1. 개요

노트 앱에 태그 기능을 추가한다. 사용자는 각 노트에 자유 형식의 문자열 태그를 붙이고 제거할 수 있으며, 태그는 노트 에디터 화면에서 인라인으로 관리된다.

**목표:**
- 노트에 태그를 부여해 맥락 정보를 빠르게 식별할 수 있게 한다.
- 태그 추가/삭제가 별도 저장 액션 없이 즉시 반영되도록 한다.

**범위:** 태그의 CRUD(추가·삭제·조회). 필터링·검색은 Out of Scope.

---

## 2. 사용자 스토리

### US-01. 태그 추가
> As a 노트 작성자,
> I want to 노트에 태그를 입력하고 Enter로 추가할 수 있도록,
> So that 해당 노트의 주제나 카테고리를 빠르게 표시할 수 있다.

**인수 조건:**
- `NoteEditor` 내 제목 아래, 본문 위에 태그 입력창이 표시된다.
- 텍스트 입력 후 `Enter`를 누르면 태그가 칩(pill) 형태로 추가된다.
- 추가 즉시 `PATCH /notes/:id`로 서버에 반영된다.
- 입력창은 추가 후 초기화된다.
- 빈 문자열(공백만 있는 경우 포함)은 무시된다.
- 20자를 초과하는 입력은 20자로 제한된다.

### US-02. 태그 삭제
> As a 노트 작성자,
> I want to 추가된 태그를 제거할 수 있도록,
> So that 잘못 입력했거나 더 이상 유효하지 않은 태그를 정리할 수 있다.

**인수 조건:**
- 태그 칩에 hover 시 `×` 버튼이 나타난다.
- `×` 클릭 시 해당 태그가 즉시 삭제되고 서버에 반영된다.
- hover 상태가 아닐 때는 `×` 버튼이 표시되지 않는다.

### US-03. 태그 목록 조회
> As a 노트 독자,
> I want to 노트 상세 화면에서 붙여진 태그 목록을 볼 수 있도록,
> So that 노트의 주제를 한눈에 파악할 수 있다.

**인수 조건:**
- 태그가 있는 노트를 열면 제목 아래에 태그 칩들이 가로로 나열된다.
- 태그가 없으면 칩 없이 입력창만 표시된다.

### US-04. 중복 태그 방지
> As a 노트 작성자,
> I want to 이미 있는 태그를 다시 추가하려 할 때 조용히 무시되도록,
> So that 노트에 동일 태그가 중복으로 쌓이지 않는다.

**인수 조건:**
- 동일한 문자열(대소문자 구분, 완전 일치) 태그는 추가되지 않는다.
- 에러 메시지나 별도 피드백 없이 조용히 무시된다.

## 4. Out of Scope

이번 릴리스에서 **의도적으로 제외**하는 항목들이다. 미래 확장 시 별도 PRD를 작성한다.

### 태그 기반 필터링 / 검색
- 사이드바 또는 별도 뷰에서 특정 태그가 붙은 노트만 필터링하는 기능
- 태그명으로 노트를 검색하는 기능
- 이유: 현재 요구사항은 노트 단위 태그 관리에 한정. 필터링은 별도 탐색 기능 설계 필요

### 전역 태그 관리
- 앱 전체에서 사용된 모든 태그 목록을 별도 화면에서 조회·수정·삭제하는 기능
- 태그 rename (여러 노트에 걸쳐 일괄 수정)
- 이유: 태그를 독립 엔티티로 관리하는 데이터 모델 전환이 선행되어야 함

### 태그 자동완성 / 추천
- 기존 사용된 태그를 드롭다운으로 제안하는 기능
- 이유: 전역 태그 목록 조회 기능이 없으므로 구현 불가. 별도 API 또는 클라이언트 캐시 설계 필요

### 태그 색상 / 아이콘 커스터마이징
- 태그별 색상을 지정하거나 시각적 스타일을 변경하는 기능
- 이유: 현재 스펙은 단순 문자열 태그. 스타일 메타데이터 저장 구조 변경 필요

### 태그 정렬 / 순서 변경
- 태그 칩의 순서를 드래그 앤 드롭으로 변경하는 기능
- 이유: 현재 `string[]` 배열 순서는 입력 순서 기준. 명시적 순서 관리는 별도 설계 필요

### 다중 태그 일괄 삭제
- 여러 태그를 동시에 선택해 한 번에 삭제하는 기능
- 이유: 현재 UX는 칩 단위 개별 삭제. 다중 선택 인터랙션은 별도 설계 필요

### 태그 입력 시 실시간 유효성 피드백
- 중복 태그 입력 시 시각적 경고 표시
- 20자 초과 시 에러 메시지 표시
- 이유: 현재 스펙은 조용히 무시(silent fail). 피드백 UX 추가 시 별도 결정 필요

### 노트 목록(NoteItem)에서의 태그 표시
- 사이드바 노트 카드(`NoteItem`)에 태그를 미리보기로 표시하는 기능
- 이유: 현재 `NoteItem`은 제목·내용 미리보기·날짜만 표시. 레이아웃 변경 필요

## 5. 용어 정의

| 용어 | 정의 |
|---|---|
| **태그(Tag)** | 노트에 붙이는 자유 형식의 문자열 레이블. 최대 20자, 대소문자 구분 |
| **태그 칩(Tag Chip)** | UI에서 태그를 표시하는 pill 형태의 컴포넌트. Knowledge Token이라고도 함 |
| **즉시 저장(Immediate Persist)** | 사용자 액션(추가/삭제) 발생 즉시 별도 저장 버튼 없이 서버에 반영되는 저장 방식 |
| **중복 태그** | 이미 해당 노트에 존재하는 것과 완전히 동일한 문자열(대소문자 구분)의 태그 |
| **silent fail** | 잘못된 입력이나 중복 등 오류 상황에서 사용자에게 별도 피드백 없이 조용히 무시하는 동작 |
| **PATCH /notes/:id** | JSON Server에서 노트의 특정 필드만 부분 업데이트하는 API 엔드포인트 |
| **NoteEditor** | 노트 생성/수정 폼 컴포넌트. 태그 UI가 위치하는 화면 |
| **전역 태그** | 특정 노트에 종속되지 않고 앱 전체에서 독립적으로 관리되는 태그 엔티티 (현재 스펙 외) |
